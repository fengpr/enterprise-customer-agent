# 企业智能客服工单 Agent 平台 PRD 摘要

## 产品定位

本项目建设 Web 管理后台、客服工作台和 Agent 服务接口，覆盖客户咨询、知识库问答、订单查询、工单流转、人工审核和运营分析。

## MVP 范围

- 登录与角色权限：访客、人工客服、客服主管、知识库管理员、系统管理员、Agent 服务。
- 智能客服对话：用户提问、AI 回复、上下文分析、人工接管。
- 意图识别：`intent` 表示业务域，覆盖咨询、物流、退款、换货、维修、投诉、发票、会员、其他。
- 用户目的识别：`user_goal` 表示用户真实目的，覆盖规则咨询、状态查询、发起申请、投诉、争议、普通信息查询、其他。
- 结构化抽取：情绪、订单号、商品名、优先级、置信度、业务域、用户目的、业务动作、动作槽位、缺失槽位和下一步动作。
- RAG 问答：仅检索已发布且未失效文档，回答必须返回引用来源。
- Agent 工具：查询订单、物流、客户工单和知识库，创建工单；自动派发仅在低风险且配置允许时触发。
- 工单管理：创建、分派、处理、转派、升级、关闭、重开。
- 业务动作闭环：退货、换货、维修、发票、取消订单、投诉提交等真实动作必须先补齐必填槽位；槽位齐全且权限校验通过后创建工单或调用受控业务 Tool。
- 派单策略：Agent 默认只创建工单，Java 工单状态保持 `PENDING_ASSIGN` 并进入调度队列；自动派单必须显式配置开启，并且仅限低风险、规则明确、非真实业务动作的工单。
- 调度确认：高风险、投诉、争议、退款/退货申请、赔付诉求和低置信度工单即使开启自动派单配置，也必须由调度员确认分派；调度台提供手动分派和调度员主动触发的智能派单。
- 人工审核：真实业务动作、赔付承诺、投诉争议、低置信度、知识库或业务工具无依据时必须转人工或进入工单；规则咨询和只读状态查询可自动回复。
- 数据分析：会话量、工单量、转人工率、知识库命中率、SLA 超时率。

## 核心流程

1. 访客发送问题，系统创建会话。
2. Agent 进行意图识别、用户目的识别、情绪识别和动作槽位抽取。
3. 如为业务动作请求，合并同会话 `pending_action_request`，计算缺失槽位和 `next_action`。
4. 如包含订单信息或前端选中订单，调用业务系统订单查询工具校验归属。
5. 调用知识库检索，过滤低相似度、下线和过期文档。
6. Agent 根据风险和槽位状态决定自动回复、追问补充信息、转人工或创建工单。
7. 记录会话、消息、pending 状态、工具调用、知识库引用和工单流转日志。

## 高风险规则

- AI 不得承诺退款成功、赔付金额、处罚商家或法律结论。
- 真实业务动作不能只返回规则建议；信息缺失时必须追问，信息齐全后进入工单或受控 Tool。
- 退款申请、赔付、投诉、举报、维权、强烈不满和售后争议必须进入工单或人工审核。
- 退款到账时效、换货规则、维修周期、发票流程和会员权益等规则咨询，可在有知识库依据时自动回复。
- 无知识库命中或低置信度回复不得自动发送。
- 工具调用必须受白名单和权限控制，所有调用必须记录日志。
- 自动派单必须显式配置开启，且仅限低风险、规则明确的工单；退款/退货申请、投诉、争议和低置信度场景不得自动派单。

## 技术约束

- `business-service`：模拟既有 Java 业务系统，提供订单、用户、商品、工单 REST 接口。
- `ai-agent-service`：Python 3.10+、FastAPI、LangChain、LangGraph、RAG、工具调用。
- 数据库：Demo 可使用 SQLite，正式环境建议 MySQL 8.0。
- 向量库：Chroma 或 FAISS。
- 前端 Demo：Streamlit，后续可替换为 Vue3 管理后台。
