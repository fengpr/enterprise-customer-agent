# 高并发部署与压测

## 运行角色

- `agent-api`：只处理在线 HTTP/SSE、鉴权、限流和 Agent 请求。
- `agent-worker`：从 Redis 队列执行真实 RAG、LLM 与业务工具调用，不暴露公网 HTTP。
- `evaluation-worker`：独立消费 DeepEval 任务，绝不部署在 API Pod 内。
- PostgreSQL：生产会话、消息、Trace、评测与 RAG 元数据存储；Redis：跨 Pod 限流、缓存和队列状态。

## 环境变量

`REDIS_URL` 必须在多副本环境配置。`AGENT_MAX_CONCURRENCY` 默认 20，`AGENT_PER_SUBJECT_CONCURRENCY` 默认 2；应按模型供应商并发配额调整。

生产环境设置 `AGENT_EXECUTION_QUEUE_ENABLED=true`：`/api/agent/reply/stream` 会发出 `accepted`、`queued`、`generating`、`completed` 或 `degraded` 事件，并由 `agent-worker` 完成真实执行。队列等待超过 30 秒会返回安全降级话术。

## 压测

先设置 `AGENT_BASE_URL` 与测试客户 `CUSTOMER_TOKEN`，执行：

```bash
k6 run load-tests/agent-reply.js
```

验收关注 API P95、429/503 占比、`/metrics` 中的降级次数，以及模型、Java 服务和数据库的资源使用率。真实 LLM 压测必须使用压测专用密钥和预算，避免消耗生产配额。
